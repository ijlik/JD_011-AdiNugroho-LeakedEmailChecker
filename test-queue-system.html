Test Case #10: Queue System with Rate Limiting

⚠️ Rate Limiting Configuration:

  • API Limit: 10 requests per minute
  • Queue Behavior: All emails scanned, processed 10 per minute
  • Real-time Updates: Popup shows live status of each email
  • Status Indicators: Pending (⏳), Safe (✅), Leaked (⚠️), Error (❌)

Testing Instructions:

  1. Open Chrome extension popup and scan this page
  2. Observe that all 15 emails are immediately shown as "Pending"
  3. Watch real-time updates as emails are processed (10 per minute)
  4. Verify status changes from pending to completed states
  5. Check console for queue processing logs
  6. Test popup persistence by closing and reopening

Real-time Status Monitoring

The popup should show:

  • ⏳ Queued for checking... - Initially for all emails
  • ✅ No known leaks - For clean emails
  • ⚠️ X breach(es) found - For compromised emails
  • ❌ Error: message - For failed requests
Expected: Status updates happen in real-time as each email is processed. Progress counter shows "X/15 completed".

Queue Progress Indicators

Monitor these behaviors:

  • Initial scan finds all 15 emails immediately
  • All emails show as "Pending" with queue status
  • First 10 emails processed within ~60 seconds
  • Remaining 5 emails processed in second minute
  • Progress counter updates: "10/15 completed", "15/15 completed"
  • Final status: "Done. All 15 emails checked."
Expected: Smooth queue progression with clear progress indicators and no API errors due to rate limiting.

Persistence Testing

During queue processing:

  1. Close the popup while emails are being processed
  2. Reopen the popup after a few seconds
  3. Verify it shows current progress state
  4. Confirm real-time updates continue
Expected: Popup reopens showing current processing state with accurate progress counter and continues receiving updates.
